# SOLID Principles trong lập trình hướng đối tượng (OOP) dễ hiểu nhất

SOLID gồm 5 nguyên lý giúp code OOP dễ bảo trì:

1. **Single Responsibility**: Mỗi class chỉ nên có 1 lý do để thay đổi.
2. **Open/Closed**: Mở rộng được mà không sửa code cũ.
3. **Liskov Substitution**: Thay thế class cha bằng class con mà không lỗi.
4. **Interface Segregation**: Interface nhỏ, chuyên biệt.
5. **Dependency Inversion**: Phụ thuộc vào abstraction, không phụ thuộc cụ thể.

Áp dụng SOLID giúp code sạch, dễ test và mở rộng.
